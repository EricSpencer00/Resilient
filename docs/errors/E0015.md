# E0015 — Import target not found
{: .no_toc }

A `use`/`import` path doesn't resolve to a module or package on
the search path.
{: .fs-5 .fw-300 }

---

## What triggers it

The module resolver looks up the path named in an `import`/`use`
statement against the local file tree and the package manifest's
dependencies, and finds neither.

## Minimal example

```resilient
import does_not_exist;
```

Output:

```text
scratch.rz:1:8: error[E0015]: cannot find module `does_not_exist`
import does_not_exist;
       ^^^^^^^^^^^^^^
```

## Fix

Check the spelling, confirm the file exists relative to the
importing file, or add the missing dependency to the package
manifest (see `rz pkg`).

## Source

Emitted from the module resolver in `resilient/src/imports.rs`.
