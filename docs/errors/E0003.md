# E0003 — Unclosed delimiter
{: .no_toc }

A `(`, `[`, or `{` was opened but the parser reached EOF before
finding its matching close.
{: .fs-5 .fw-300 }

---

## What triggers it

Every opening delimiter must have a matching close at the same
nesting level. When one doesn't, the parser reports E0003
pointing at the unclosed opener.

## Minimal example

```rs
fn main() {
    let xs = [1, 2, 3;
}
```

Output:

```text
scratch.rs:2:14: error[E0003]: unclosed `[`
   let xs = [1, 2, 3;
            ^
```

## Fix

Close every delimiter you open. Editors with bracket-pair
highlighting make this easy to spot.

```rs
fn main() {
    let xs = [1, 2, 3];
}
```
